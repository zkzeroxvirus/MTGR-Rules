# **Experimental Seasonal Event Integration V0.1**

Seasonal Event Integration introduces limited-time Event content into regular runs rather than placing it in a separate curated run.

During an active season, players may randomly encounter Event Encounters, Event cards, and Event Cash Outs while following the normal run structure.

## **Purpose**

This model allows a season to influence ordinary play without requiring players to select a separate Event Run.

Because seasonal content is encountered randomly, an integrated season is expected to remain active for an extended period. The current proposal is approximately one quarter of the year.

## **Seasonal Content**

An integrated season may add limited-time content to one or more regular game pools:

* **Event Encounters** — Seasonal Host decks and encounter rules
* **Event Cards** — In-run effects that introduce or interact with seasonal content
* **Event Cash Outs** — Seasonal rewards added to eligible Cash Out results
* **Event Crypts** — Seasonal Crypt encounters, if enabled for that Event
* **Seasonal Rewards** — Currency, loot, XP, or other rewards defined by the Event

The exact content added to each pool is defined separately for each season.

## **Event Encounter Access**

Event Encounters appear through seasonal effects during a regular run.

An Event card may use wording such as:

> **The next encounter is an Event Encounter. Draw another Event.**

When this effect resolves:

* The next scheduled encounter is selected from the active season's Event Encounter pool
* Drawing another Event allows the current Event step to continue rather than replacing its entire result with the encounter change
* After the Event Encounter is completed, the run returns to its normal structure unless another effect says otherwise

Whether the Event Encounter replaces the next encounter or is inserted as an additional encounter remains under discussion.

## **Event Deck Selection**

Each seasonal encounter slot may contain several eligible Event decks. When an Event Encounter is triggered, choose its deck using the selection method defined by that season.

Possible methods for testing include:

* Select one Event deck completely at random
* Reveal several Event decks and use the standard player-elimination and Host-choice process
* Select from a pool based on the current encounter number or party progression

## **Event Cash Outs**

Seasonal Cash Outs may appear among normal Cash Out results for the duration of the Event.

A seasonal Cash Out may grant Event-specific loot, currency, progression, or access to additional Event content. Unless its rules say otherwise, it follows the normal timing and resolution rules for Cash Outs.

## **Event Crypts and Event Crypt Buffs**

An integrated season may include multiple Event Crypts. All Event Crypts belonging to the same season provide the same limited-time **Event Crypt Buff** when defeated.

The method for reaching an Event Crypt is not yet determined. Options include:

* Randomly replacing the regular Crypt
* Unlocking an Event Crypt through Event cards or seasonal progression
* Allowing the party to choose between the regular Crypt and an unlocked Event Crypt

The Buff's effect, recipients, duration, and repeat-completion behavior must be defined by the active season.

## **Season Duration**

The current proposal is for each integrated Seasonal Event to remain available for approximately three months.

The longer duration is intended to give players multiple opportunities to encounter randomly distributed seasonal content. Exact start and end dates must be announced for each Event.

## **Run Compatibility**

Regular run structure remains active unless a seasonal rule explicitly changes it.

* Normal Encounters remain in their pools
* Seasonal content does not guarantee that every run contains an Event Encounter
* After resolving seasonal content, play returns to the normal run sequence
* Seasonal rules should identify any content that cannot appear together

## **Current Test Variables**

The following rules require discussion or playtesting:

* How frequently seasonal Events, Encounters, and Cash Outs should appear
* Whether an Event Encounter replaces or adds to the scheduled encounter
* Whether players should be guaranteed to see seasonal content after a certain number of runs
* How Event decks are selected
* How an Event Crypt is discovered or unlocked
* Whether the Event Crypt Buff is awarded to the party or each participating player
* Whether repeat Event Crypt victories refresh or improve the Buff
* Whether three months is the correct availability window
* What happens when a run remains active after the season ends

## **Status**

Seasonal Event Integration is an experimental proposal and is not part of the core Event or Encounter systems. Each season must define its active content pools, appearance rates, Event Crypt access, Event Crypt Buff, availability window, and special reward rules before play begins.
