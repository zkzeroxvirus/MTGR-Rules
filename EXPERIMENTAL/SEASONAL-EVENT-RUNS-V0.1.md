# **Experimental Seasonal Event Runs V0.1**

Seasonal Event Runs are limited-time, curated runs built entirely around a seasonal theme.

Each Event Run contains a fixed sequence of Event Encounters and culminates in an Event Crypt. Multiple eligible decks are available at every stage so repeated runs can produce different combinations while retaining the same seasonal structure.

## **Purpose**

This model gives players a reliable way to experience the featured seasonal content from beginning to end.

Seasonal Event Runs are separate from regular runs unless an Event later states otherwise.

## **Event Run Structure**

A standard Seasonal Event Run contains:

1. **Event Encounter 1**
2. **Event Encounter 2**
3. **Event Encounter 3**
4. **Event Crypt**

The encounters are resolved in this order. Each stage uses its own curated pool of Event decks.

| Stage | Content |
| ----- | ----- |
| Event Encounter 1 | One deck from the Event Encounter 1 pool |
| Event Encounter 2 | One deck from the Event Encounter 2 pool |
| Event Encounter 3 | One deck from the Event Encounter 3 pool |
| Event Crypt | One Event Crypt from the seasonal Crypt pool |

## **Event Deck Selection**

Each encounter stage contains several eligible Event decks rather than one fixed deck.

The deck used for a stage is selected when that encounter begins. This allows different Event Runs during the same season to feature different opponents.

The final selection method is not yet determined. Options for testing include:

* Select one deck completely at random
* Reveal several decks and use the standard player-elimination and Host-choice process
* Prevent a deck from repeating until every deck in its stage pool has appeared

## **Event Crypts**

The final encounter is selected from a curated pool of Event Crypts.

* Each Event Crypt may use a different deck, theme, or encounter mechanic
* Every Event Crypt associated with the same season grants the same **Event Crypt Buff**
* The party does not choose a weaker or stronger Buff based on which Event Crypt is selected

## **Event Crypt Buff**

The Event Crypt Buff is a limited-time reward associated with the active season.

The Buff is granted when the party defeats an Event Crypt. Its specific effect and duration are defined by that seasonal event.

The following details remain undecided:

* Whether the Buff applies to the party, individual players, or the entire community
* Whether the Buff lasts until the end of the season or for a fixed number of runs
* Whether another Event Crypt victory refreshes or improves the Buff
* Whether the Buff may be earned more than once
* What happens to an active Buff when the season ends

## **Rewards and Progression**

Unless a Seasonal Event defines replacements, Event Encounters use the normal encounter victory, loss, reward, and post-encounter procedures.

Any additional seasonal currency, XP, loot, or completion rewards must be listed in that Event's rules.

## **Current Test Variables**

The following rules require discussion or playtesting:

* Whether a Seasonal Event Run should always contain exactly three Event Encounters
* How Event decks and Event Crypts are selected
* Whether failed Event Runs may be resumed or must restart
* Whether regular Events, Town visits, and Stay Out choices occur during an Event Run
* Whether the Event Crypt Buff is awarded to the party or each participating player
* How repeat completion is rewarded
* Whether Seasonal Event Runs coexist with or temporarily replace regular runs

## **Status**

Seasonal Event Runs are an experimental proposal and are not part of the core game structure. Each season must define its encounter pools, Event Crypt pool, Event Crypt Buff, availability window, and any special reward rules before play begins.
